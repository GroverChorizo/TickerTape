## **3. Funding rate extreme** - 
## What Are Funding Rate Extremes?

**Funding rate extremes** occur when the periodic fee paid between long and short traders in perpetual futures markets reaches unusually high or low levels, signaling **overheated bullish or bearish sentiment**.  These extremes often precede **reversals, squeezes, or liquidation cascades**, as leveraged traders face mounting costs and exit positions. 

### On Hyperliquid:
- Funding is paid **hourly**. 
- Formula:  
  `Funding Rate = Average Premium Index + clamp(0.000125 - Premium Index, -0.0005, 0.0005)`  
  (Fixed interest rate = 0.01% per 8h → ~0.00125% per hour)
- Capped at **±4% per hour** (extremely rare). 
- **Positive rate**: Longs pay shorts → bullish bias. 
- **Negative rate**: Shorts pay longs → bearish bias.

### Defining "Extreme":
| Level | Funding Rate (Hourly) | Interpretation |
|------|------------------------|----------------|
| Normal | < ±0.005% | Balanced market |
| Elevated | ±0.005% – ±0.01% | Moderate bias |
| **Extreme** | **> ±0.01%** | **High leverage, potential reversal risk** |

Example: A rate of **+0.012%** means longs pay **$12/hour per $100k notional** — unsustainable in a sideways or correcting market.

## How to Monitor & Alert in Python Using Hyperliquid API

Use the official **Hyperliquid Data Layer API** to fetch real-time funding rates and trigger alerts in your Textualize dashboard. 

### Step 1: Fetch Live Funding Rates

    ```python
    import httpx
    import asyncio
    from typing import Dict

    async def fetch_hyperliquid_funding(coin: str = "BTC") -> Dict:
        url = "https://api.hyperliquid.xyz/info"
        payload = {
            "type": "fundingHistory",
            "coin": coin,
            "startTime": int((asyncio.get_event_loop().time() - 3600) * 1000)  # Last hour
        }
        headers = {"Content-Type": "application/json"}
        
        async with httpx.AsyncClient() as client:
            resp = await client.post(url, json=payload, headers=headers)
            data = resp.json()
            if data and len(data) > 0:
                latest = data[-1]
                return {
                    "coin": latest["coin"],
                    "rate": float(latest["fundingRate"]),
                    "time": latest["time"]
                }
        return None
    ```

    ```python
                label.update(f"[bold red]ALERT: {self.funding_rate:+.4f}%[/]")
            else:
                self.alert_active = False
                label.update(f"Normal: {self.funding_rate:+.4f}%")
    ```

### Step 1: Fetch Real Data (No Synthetics)

```python
import httpx
import asyncio
from datetime import datetime

async def fetch_anomalies(coin: str = "BTC"):
    url = "https://api.hyperliquid.xyz/info"
    headers = {"Content-Type": "application/json"}

    async with httpx.AsyncClient() as client:
        # Get latest candle (5m)
        candle_resp = await client.post(url, json={
            "type": "candleSnapshot",
            "req": {"coin": coin, "interval": "5m", "startTime": int((datetime.now().timestamp() - 300) * 1000)}
        }, headers=headers)
        candles = candle_resp.json()

        # Get funding rate
        funding_resp = await client.post(url, json={
            "type": "fundingHistory",
            "coin": coin,
            "startTime": int(datetime.now().timestamp() * 1000)
        }, headers=headers)
        funding = funding_resp.json()

        # Get order book
        book_resp = await client.post(url, json={
            "type": "l2Book",
            "coin": coin
        }, headers=headers)
        book = book_resp.json()

        return {
            "price": float(candles[-1]["h"]) if candles else None,
            "volume": float(candles[-1]["v"]) if candles else 0,
            "funding_rate": float(funding[-1]["fundingRate"]) if funding else 0,
            "bid_depth": sum(float(level[1]) for level in book["levels"][0][:3]),
            "ask_depth": sum(float(level[1]) for level in book["levels"][1][:3])
        }
```

### Step 2: Define Anomaly Logic

```python
def detect_anomaly(data, history):
    alerts = []

    # Price spike (5m vs 1h MA)
    short_ma = data["price"]
    long_ma = sum(h["price"] for h in history[-12:]) / 12  # 1h
    if abs(short_ma - long_ma) / long_ma > 0.05:
        alerts.append("🚨 PRICE SPIKE")

    # Volume surge
    avg_volume = sum(h["volume"] for h in history[-12:]) / 12
    if data["volume"] > 3 * avg_volume:
        alerts.append("🚨 VOLUME SURGE")

    # Funding extreme
    if abs(data["funding_rate"]) > 0.0001:
        alerts.append("🚨 EXTREME FUNDING")

    # Liquidity drop
    prev_bid, prev_ask = history[-1]["bid_depth"], history[-1]["ask_depth"]
    if prev_bid and (abs(data["bid_depth"] - prev_bid) / prev_bid > 0.5):
        alerts.append("🚨 BID LIQUIDITY DRAIN")
    if prev_ask and (abs(data["ask_depth"] - prev_ask) / prev_ask > 0.5):
        alerts.append("🚨 ASK LIQUIDITY DRAIN")

    return alerts
```

## **5. HLP z-score threshold** - 
## What Are HLP Z-Score Thresholds?

**HLP (Hyperliquid Pool) z-score thresholds** measure how many standard deviations the current funding rate is from its historical mean, indicating **extreme market imbalance**. A high z-score suggests **over-leveraged longs or shorts**, increasing the risk of a **reversal or liquidation cascade**. 

### Formula:
```
z = (current_funding_rate - mean_rate) / std_dev_rate
```

### Thresholds (Based on Hyperliquid Data):
| Z-Score | Interpretation |
|--------|----------------|
| **|z| > 2.0** | **Anomaly Alert**: Unusual funding pressure |
| **|z| > 3.0** | **Extreme Alert**: High probability of reversal or squeeze |

> Example: If BTC funding averages +0.003%/hr (σ = 0.001%), a rate of +0.006% → z = 3.0 → **extreme long bias**.

**Implementation:**
- Use Unicode Braille characters (⠀⡀⢀⣀) for 8-point vertical resolution
- Color: Cyan for price above MA, Red for below
- Length: 16-24 characters (last 24 periods)
- Update: Real-time with new candle close

**Pattern Recognition:**
- **Uptrend:** Ascending Braille pattern
- **Downtrend:** Descending pattern
- **Consolidation:** Flat horizontal pattern
- **Breakout:** Sharp vertical move in recent bars

**Best Libraries:**
```python
from rich.console import Console
from rich.sparkline import Sparkline  # For sparklines

# Or custom implementation:
def create_sparkline(data: List[float], width: int = 20) -> str:
    """Generate Unicode Braille sparkline"""
    # Normalize to 0-7 range for Braille characters
    normalized = [(x - min(data)) / (max(data) - min(data)) * 7 for x in data]
    braille_base = 0x2800  # Base Braille Unicode
    
    sparkline = ""
    for val in normalized[-width:]:
        sparkline += chr(braille_base + int(val))
    
    return sparkline
```

## 🎨 **Unified Design Principles**

### **Color Harmony Rules**

1. **Directional Data:**
   - Green: Bullish, buying, positive
   - Red: Bearish, selling, negative
   - Gray: Neutral, inactive

2. **Intensity Encoding:**
   - Brighter = More recent/important
   - Darker = Older/less significant
   - Flashing = Alert condition

3. **Accent Colors (Sparingly):**
   - Cyan: Headers, important labels
   - Magenta: Coin tickers, identifiers
   - Purple: AI insights, special features

4. **Background Contrast:**
   - Very dark background (#0a0e1a)
   - Panel background slightly lighter (#0d1117)
   - Data on lighter gray (#1c2128)
   - Text on white (#ffffff)

### **Layout Flow**

```
┌─────────────────────────────────────────────┐
│ Top: High-level overview (Market Stats)     │← Sparklines
├─────────────────────────────────────────────┤
│ Left: Sentiment & Flow Analysis             │← Gauges, Bars
│ Right: Orderbook & Liquidity                │← Depth Charts
├─────────────────────────────────────────────┤
│ Bottom: Detail panels (Whales, Liqs, etc.)  │← Tables, Heatmaps
└─────────────────────────────────────────────┘
```

**Information Hierarchy:**
1. **Glanceable (Sparklines, Gauges):** Quick trend check
2. **Scannable (Tables, Bars):** Identify anomalies
3. **Detailed (Heatmaps, Charts):** Deep pattern analysis

```python
async def show_cascade_alert(self, liquidations):
    # Show banner immediately
    self.banner.update("⚠️ CASCADE ALERT ⚠️")
    self.banner.add_class("alert-danger")
    
    # Stagger fade-in for each liquidation
    for i, liq in enumerate(liquidations):
        await asyncio.sleep(0.1)  # 100ms delay
        row = self.create_liq_row(liq)
        row.styles.opacity = 0
        self.table.add_row(row)
        
        # Animate opacity 0 → 1 over 200ms
        row.styles.animate("opacity", value=1.0, duration=0.2)
```

**Why Fade-In?**
- **Reduces jarring jumps** when multiple items appear
- **Draws attention** to new critical events
- **Creates visual hierarchy** (newer = more important)

```python
def update_price(self, asset, old_price, new_price):
    # Animate number transition
    self.price_label.update(f"{new_price:,.0f}")
    
    # Show direction indicator
    if new_price > old_price:
        self.arrow.update("▲")
        self.arrow.add_class("price-up")  # Green
    else:
        self.arrow.update("▼")
        self.arrow.add_class("price-down")  # Red
    
    # Fade out arrow after 1s
    self.set_timer(1.0, lambda: self.arrow.update(""))
```

**Direction Indicators:**

- **● (Yellow):** Unchanged (for reference)
- **▲▲▲ (Bright Green):** Large increase (>1%)
- **▼▼▼ (Bright Red):** Large decrease (>1%)

```python
class CommandAutocomplete:
    """Intelligent autocomplete with context awareness"""
    
    COMMANDS = {
        '/whales': {
            'description': 'View whale trades',
            'args': ['btc', 'eth', 'sol', 'hype', 'all'],
            'flags': {
                '--min': 'Minimum trade size (USD)',
                '--timeframe': 'Time period (1h, 4h, 1d)',
                '--type': 'Filter by buy/sell',
                '--export': 'Export to CSV'
            }
        },
        '/backtest': {
            'description': 'Run strategy backtest',
            'args': ['run', 'resume', 'report'],
            'flags': {
                '--data': 'Path to CSV data file',
                '--strategy': 'Strategy name',
                '--capital': 'Initial capital amount',
                '--monte-carlo': 'Run Monte Carlo (N sims)'
            }
        },
        # ... more commands
    }
    
    def get_suggestions(self, partial_input: str) -> List[str]:
        """
        Return autocomplete suggestions based on current input.
        
        Logic:
        1. If starts with '/' → suggest commands
        2. If command complete → suggest args
        3. If has '--' → suggest flags
        """
        tokens = partial_input.split()
        
        if len(tokens) == 0 or not tokens[0].startswith('/'):
            # No command yet, suggest all commands
            return [cmd for cmd in self.COMMANDS.keys()]
        
        command = tokens[0]
        
        if command not in self.COMMANDS:
            # Unknown command, fuzzy match
            return self.fuzzy_match_commands(command)
        
        # Check if typing flag
        if len(tokens) > 1 and tokens[-1].startswith('--'):
            return list(self.COMMANDS[command]['flags'].keys())
        
        # Suggest arguments
        if len(tokens) == 1:
            return self.COMMANDS[command]['args']
        
        return []
    
    def fuzzy_match_commands(self, partial: str) -> List[str]:
        """Fuzzy matching for typos"""
        from difflib import get_close_matches
        
        matches = get_close_matches(
            partial, 
            self.COMMANDS.keys(), 
            n=3, 
            cutoff=0.6
        )
        return matches
```
